SVG

Scalable Vector Graphics

Hello friends, I want to speak about SVG in this presentation

It is an XML-based vector image format for defining two-dimensional graphics, having support for interactivity and animation.

SVG structure is based on XML. SVG format adds possibility for interaction and animation

The SVG specification was developed back in 1999 by the World Wide Web Consortium.

Early adoption was limited due to lack of support in older versions of Internet Explorer.

Since 2011, all major desktop browsers began to support SVG.

Benefits of SVG compared to CSS

  1. More complex shapes
  2. Better control of shape elements
  3. Can position text along a curve
SVG allows much more complex shapes than basic squares and circles of css It is more easy to resize and control an element composed of several basic shapes with SVG than with CSS You can position a string of text along a curve

Icon font and Object

While using SVG as image and inline SVG are pretty obvious and common, let me pay more attention to image font and object options.

Icon font

  • Icon fonts are typefaces that use images rather than letter forms.
  • Like type, each character is scalable and can be modified using CSS.
  • Icon fonts are transparent by nature so you can put them on any background and you can change the opacity of icons.
  • The most popular library of icon fonts is Font Awesome. It’s used for more than 100 million websites and includes a robust collection of icons.

Cons of Using Icon Fonts

  • You can't apply gradient to it.
  • You might not need a full icon set.
  • Some screen readers don’t know what to do with icon fonts.
  • You might not find a pack that has all icons you need.

Object

Using SVG as object gives pretty good control over animation and modification of SVG image. However there is a pitfall that you can't assign Event Listener to this object element.
